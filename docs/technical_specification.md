## 1. Общая информация о проекте

### 1.1 Описание компании

Компания специализируется на проектировании и монтаже инженерных систем для загородных домов: отопление, вентиляция, водоснабжение, электрика, канализация. Штат — ~20 человек, из них 12 инженеров-проектировщиков. Отдела продаж в актуальной структуре нет. Заявки с сайта gkproject.ru обрабатывает один из собственников компании по мере возможности.

**Текущий процесс:** потенциальные клиенты переходят на сайт из рекламных объявлений на Яндекс и Avito, оставляют заявки на сайте или звонят. Входящий поток — 3–5 обращений в день. 80% вопросов — типовые (цены, мощность оборудования, сроки монтажа).

### 1.2 Проблема

В условиях отсутствия отдела продаж на входящие запросы от потенциальных клиентов отвечает собственник, что не является его функциональной обязанностью. Собственник тратит до 40% рабочего времени на ответы на однотипные вопросы вместо работы со сложными проектами и управления компанией.

Клиенты не получают ответы в нерабочее время, часть лидов теряется (до 30%). Вся необходимая информация есть на сайте, но клиенты не хотят искать самостоятельно, а менеджеры не успевают обрабатывать все заявки.

Менеджер-консультант способен обработать 10–15 заявок в день, при этом 30% лидов теряется из-за долгого ожидания ответа. Компания готова нанять такого сотрудника с окладом ~85 000 руб./мес ФОТ. Вопросы типовые, их может решать AI.

## 2. Цели и задачи

### 2.1 Главная цель проекта

Создать ИИ-ассистента, который будет автоматически консультировать клиентов по инженерным системам 24/7, квалифицировать потенциального клиента (тёплый / холодный) и собирать заявки на консультацию с инженером, передавая их в отдел продаж через Telegram.

### 2.2 Измеримые показатели успеха (KPI)

| Метрика | Целевое значение |
|---------|-----------------|
| Снижение нагрузки на команду | 60–80% типовых вопросов на AI |
| Скорость ответа пользователю | Менее 5 секунд |
| Точность ответов | 90%+ корректных ответов |
| Время реакции на заявку | С 5–30 минут до 10–30 секунд |

## 3. Пользователи и сценарии

### 3.1 Целевые пользователи

| Группа | Описание | Частота |
|--------|----------|---------|
| Потенциальные клиенты | Владельцы загородных домов (тёплый контур), обращающиеся за консультацией по инженерным системам | 3–5 обращений/день |
| Менеджеры ОП | Сотрудники компании. После внедрения AI занимаются только сложными проектами | Ежедневно |
| ТОП-менеджеры | Получают уведомления о новых заявках в Telegram, контролируют обработку лидов | По мере поступления |

### 3.2 Пользовательские сценарии (Use Cases)

**Сценарий 1: Консультация по оборудованию**

| Параметр | Описание |
|----------|----------|
| Пользователь | Потенциальный клиент |
| Триггер | Клиент пишет: «Какой котел выбрать для дома 150 кв.м?» |
| Результат | AI-агент находит информацию в базе знаний (сайт), даёт ответ с кликабельной ссылкой на страницу-источник |

**Сценарий 2: Оформление заявки на консультацию**

| Параметр | Описание |
|----------|----------|
| Пользователь | Потенциальный клиент |
| Триггер | Клиент пишет: «Хочу заказать монтаж отопления, свяжитесь со мной» |
| Результат | AI собирает имя и телефон, отправляет заявку в Telegram-группу ОП и дублирует в личные сообщения каждому менеджеру |

**Сценарий 3: Обновление базы знаний**

| Параметр | Описание |
|----------|----------|
| Пользователь | Администратор системы |
| Триггер | Запланированное задание (1 раз в месяц) |
| Результат | Парсер сканирует сайт компании, обновляет векторную базу данных ChromaDB новыми страницами |

## 4. Функциональные требования

### 4.1 Основные функции (Must Have)

- Чат-виджет на сайте для общения с клиентами (React, TypeScript)
- RAG-система для поиска ответов по базе знаний компании (ChromaDB)
- Выбор LLM: приоритет GigaChat для РФ, fallback на ChatGPT при недоступности
- Ссылка на источник в каждом ответе AI (кликабельная)
- Telegram-бот для передачи заявок в группу отдела продаж
- Автоматическая рассылка заявок в личные сообщения всем сотрудникам ОП
- Логирование всех диалогов

### 4.2 Желательные функции (Should Have)

- Автоматический парсер сайта (1 раз в месяц)
- Кнопка «Взял в работу» в Telegram для фиксации ответственного
- Админ-панель для просмотра логов и статистики

### 4.3 Возможные функции (Could Have)

- Интеграция с CRM (AmoCRM / Битрикс24)
- Аналитика по частоте вопросов и темам
- Отчёт по эффективности AI (точность ответов, количество обработанных заявок)

### 4.4 Исключения (Won't Have)

- Автоматическое оформление договоров
- Голосовые звонки через AI
- Интеграция с 1С в первой версии

### 5.1 Источники данных

| Источник | Формат | Объём | Статус |

- **Доступ:** редактирование базы знаний — через перезапуск парсера или ручное добавление

## 6. Технический стек

### 6.1 Выбранные технологии

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| LLM (основная) | GigaChat (Сбер) | Российская LLM, без VPN, соответствует 152-ФЗ, дешевле OpenAI (2–5 руб./1K токенов vs 15–20 руб.), нативная кириллица |
| LLM (резервная) | OpenAI GPT-4o-mini | Fallback при недоступности GigaChat. Качественнее на сложных запросах |
| Векторное хранилище | ChromaDB | Легковесная, встраиваемая БД. Не требует отдельного сервера. Хорошая интеграция с LangChain и Python |
| Backend | FastAPI (Python) | Асинхронный фреймворк, встроенная поддержка SSE, автогенерация OpenAPI-документации |
| Парсер данных | Python + BeautifulSoup4 | Легковесное решение для статических сайтов, минимум зависимостей |
| Frontend | React + TypeScript + Vite | Современный стек, быстрая сборка, типизация |
| Чат-виджет | Кастомный React-компонент | Встраивается на любой сайт (1 строка кода), полный контроль над UI/UX |
| База данных (логи) | SQLite / JSON-файлы | Достаточно для MVP, структурированное хранение, экспорт и анализ |
| Telegram | python-telegram-bot (v20+) | Асинхронная, поддержка webhook/polling, встроенная клавиатура |
| Деплой | Docker + Docker Compose | Изоляция сервисов, воспроизводимость, единая команда запуска |
| CI/CD | GitHub Actions (опционально) | Автодеплой на VPS при пуше в main, прогон тестов |

### 6.2 Отклонённые альтернативы

| Компонент | Альтернатива | Причина отклонения |
|-----------|-------------|-------------------|
| LLM | YandexGPT | Сложнее с получением доступа для юрлиц, менее прозрачное ценообразование |
| LLM | Только ChatGPT | VPN-зависимость, валютные риски, возможные блокировки, не соответствует 152-ФЗ |
| Векторное хранилище | Pinecone | Платное облачное решение (от $70/мес), избыточно для MVP |
| Векторное хранилище | Qdrant (self-hosted) | Требует отдельный сервер и ресурсы |
| Backend | Django | Слишком тяжеловесный для API. FastAPI легче и быстрее для асинхронных задач |
| Парсер | Scrapy | Избыточен для 50–100 страниц |
| Frontend | Vue.js / Svelte | React выбран из-за большей экосистемы и распространённости |
| Деплой | Ручной через git pull | Высокий риск ошибок. Docker обеспечивает воспроизводимость окружения |

### 6.3 Требования к окружению

| Компонент | Минимальные требования |
|-----------|----------------------|
| VPS сервер | 2 vCPU, 4 GB RAM, 40 GB SSD |
| ОС | Ubuntu 22.04 / Debian 11 |
| Docker | Версия 24.0+ |
| Docker Compose | Версия 2.20+ |
| Доступ в интернет | Для вызовов GigaChat API и OpenAI API |
| Переменные окружения | GIGACHAT_API_KEY, OPENAI_API_KEY (опционально), TELEGRAM_BOT_TOKEN |

## 7. Ограничения и риски

### 7.1 Ограничения проекта

| Параметр | Значение |
|----------|----------|
| Бюджет на API | До 15 000 руб./мес (GigaChat) |
| Сроки | MVP за 2–3 недели |
| Команда | 1 промпт-инженер / разработчик |
| Инфраструктура | Требуется VPS для деплоя, доступ к API GigaChat и OpenAI |

### 7.2 Риски и митигация

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Галлюцинации LLM | Средняя | RAG с принудительной ссылкой на источник, системный промпт запрещает выдумывать |
| Низкое качество данных | Средняя | Ручная проверка после первого парсинга, возможность ручного добавления документов |
| Превышение бюджета на API | Низкая | GigaChat дешевле ChatGPT, установлены лимиты на токены |
| Отключение OpenAI | Низкая | Fallback на GigaChat, полная автономность в РФ |
| VPN-зависимость | Низкая | GigaChat работает без VPN |
