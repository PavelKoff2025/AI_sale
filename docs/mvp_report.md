## 1. Описание проекта

AI Sale — автономный AI-агент продаж, который устанавливается на сайт компании и в режиме 24/7 консультирует посетителей, отвечает на вопросы по услугам и собирает заявки с контактами клиентов.

**Клиент:** ООО «ГК Проект» — инженерно-строительное бюро, специализирующееся на проектировании и монтаже инженерных систем для загородных домов и коммерческих объектов в Москве и области.

**Проблема:** Потенциальные клиенты заходят на сайт в нерабочее время, не получают ответов на вопросы и уходят к конкурентам. Менеджеры тратят время на типовые консультации вместо закрытия сделок.

**Решение:** Интеллектуальный чат-виджет на базе GPT-4o с собственной базой знаний компании, голосовым вводом/выводом и автоматической передачей заявок в отдел продаж.

## 2. Архитектура системы

### 2.1 Общая схема

```
Пользователь (браузер)
    │
    ▼
Frontend (React + TypeScript + Vite)
    │  Nginx (HTTPS, proxy)
    ▼
Backend (FastAPI + Python 3.11)
    ├── RAG Engine (ChromaDB)
    ├── LLM Provider (OpenAI GPT-4o / GigaChat)
    ├── Voice (Whisper STT + TTS)
    ├── Telegram Bot (уведомления)
    └── Google Sheets (дублирование заявок)
    
Parsing Agent (BeautifulSoup + Python)
    │  Парсинг gkproject.ru
    ▼
ChromaDB (векторная база знаний)
```

## 3. Компоненты системы

### 3.1 Frontend — Чат-виджет

**Технологии:** React 18 + TypeScript + Vite + TailwindCSS

**Компоненты:**

| Компонент | Назначение |
|-----------|------------|
| ChatWidget | Корневой компонент виджета, управление состоянием открытия |
| ChatBubble | Плавающая кнопка с логотипом компании для открытия чата |
| ChatWindow | Окно чата: сообщения, быстрые ответы, форма ввода |
| Header | Шапка виджета с брендингом и кнопкой «Заявка» |
| MessageList | Прокручиваемый список сообщений с автоскроллом |
| MessageItem | Рендеринг сообщений с поддержкой markdown (жирный текст, ссылки) |
| InputBar | Поле ввода текста + кнопка микрофона + кнопка отправки |
| QuickReplies | Быстрые кнопки: «Какие услуги?», «Сколько стоит монтаж?» и др. |
| LeadForm | Форма заявки: имя, телефон (с автоформатированием) |
| VoicePlayButton | Озвучка ответов AI через OpenAI TTS |
| TypingIndicator | Анимация набора текста (три точки) |

**Хуки:**

| Хук | Назначение |
|-----|------------|
| useChat | Управление состоянием чата, SSE-стриминг ответов, AbortController |
| useSession | Сессия пользователя в localStorage, уникальный session_id |
| useVoiceRecorder | Запись голоса через MediaRecorder, распознавание через Whisper |

**Особенности реализации:**
- Server-Sent Events (SSE) для стриминга ответов в реальном времени
- Голосовой ввод через браузерный MediaRecorder API (требует HTTPS)
- Поддержка встраивания через iframe (`?embed=1`)
- Адаптивная верстка (мобильные устройства — полноэкранный режим)
- XSS-защита при рендеринге ссылок из ответов AI
- Автоматическое форматирование телефона в форме заявки

### 3.2 Backend — API-сервер

**Технологии:** FastAPI + Python 3.11 + asyncio

**API-эндпоинты:**

| Метод | URL | Назначение |
|-------|-----|------------|
| POST | /api/chat | Отправка сообщения (синхронный ответ) |
| POST | /api/chat/stream | Стриминг ответа через SSE |
| WS | /ws/chat | WebSocket чат (до 15 одновременных подключений) |
| POST | /api/voice/transcribe | Распознавание речи (Whisper) |
| POST | /api/voice/synthesize | Синтез речи (TTS → MP3) |
| POST | /api/leads/ | Создание заявки → Telegram + Google Sheets |
| GET | /api/leads/ | Список заявок (admin) |
| GET | /api/health | Проверка состояния системы |
| GET | /api/knowledge/ | Статистика базы знаний |
| DELETE | /api/knowledge/{id} | Удаление чанка из RAG (admin) |
| GET | /api/analytics/ | Аналитика: сессии, логи, статистика |

**Ключевые сервисы:**

| Сервис | Назначение |
|--------|------------|
| AgentService | Оркестрация: intent-анализ → RAG-поиск → формирование промпта → LLM-ответ |
| LLMProvider | Абстракция над LLM: OpenAI (primary) / GigaChat (fallback) / Auto (failover) |
| RAGEngine | Поиск по векторной базе: embedding запроса → ChromaDB → top-K релевантных фрагментов |
| SessionService | Управление сессиями чата в памяти (с asyncio.Lock) |
| TelegramService | Отправка заявок в Telegram-группу отдела продаж |
| GoogleSheetsService | Дублирование заявок в Google Sheets |
| VoiceService | STT (Whisper) и TTS (OpenAI) |
| ConversationLogger | Логирование разговоров, заявок, событий в JSONL |
| CacheService | TTL-кэш для повторяющихся запросов |

**Безопасность:**
- Аутентификация admin-эндпоинтов через X-Admin-Key
- Экранирование HTML в Telegram-сообщениях (html.escape)
- Валидация URL-протоколов (защита от javascript: XSS)
- Ограничение размера хранилища заявок (MAX_LEADS_STORED = 1000)
- Асинхронные блокировки (asyncio.Lock) для предотвращения race conditions

**Асинхронность:**
- Все блокирующие операции (ChromaDB, Google Sheets, файловый I/O) обёрнуты в `run_in_executor`
- Семафор на LLM-запросы (MAX_CONCURRENT_LLM = 10)
- WebSocket лимит с атомарной проверкой через asyncio.Lock

### 3.3 RAG — Retrieval-Augmented Generation

**Технологии:** ChromaDB + OpenAI Embeddings

**Принцип работы:**
1. Парсинг сайта gkproject.ru → извлечение текстового контента
2. Разбиение на чанки (~500 токенов / ~2000 символов)
3. Генерация векторных embeddings (OpenAI text-embedding-ada-002)
4. Сохранение в ChromaDB (cosine similarity)
5. При запросе пользователя: embedding запроса → поиск ближайших чанков → передача в LLM как контекст

**Особенности:**
- Косинусное расстояние для сравнения векторов
- Порог релевантности (rag_score_threshold) для фильтрации нерелевантных результатов
- TTL-кэш для повторяющихся запросов
- Fallback: если ChromaDB-сервер недоступен → локальный PersistentClient

### 3.4 Parsing Agent — Парсер базы знаний

**Технологии:** Python + BeautifulSoup4 + langchain-text-splitters

**Возможности:**
- Парсинг сайта gkproject.ru по категориям (котельные, отопление, водоснабжение, электрика, канализация)
- Поддержка форматов: веб-страницы, PDF, DOCX, Markdown
- Автоматическое обнаружение ссылок на сайте
- Очистка текста (удаление дубликатов, HTML-тегов, лишних пробелов)
- Разбиение на чанки с перекрытием (chunk_overlap = 200)
- Загрузка в ChromaDB с дедупликацией по chunk_id
- CLI-интерфейс с флагами: --parse-only, --load-only, --clear-collection, --stats

**Конфигурация через YAML:**
- `sources.yaml` — список URL и категорий для парсинга
- `processing.yaml` — параметры чанкинга, очистки, embeddings

### 3.5 Системный промпт

AI-консультант работает по строгому скрипту:

- Отвечает **только** на основе базы знаний (RAG) — не выдумывает факты
- В конце каждого ответа даёт ссылку на источник
- **Не утверждает** что заявка передана — сначала спрашивает согласие клиента
- Предлагает заполнить форму заявки (имя, телефон) через виджет
- Вежливо отклоняет вопросы не по теме
- Не обсуждает зарплаты, конкурентов, не даёт медицинских/юридических советов

## 5. Проделанная работа

### Этап 1: Аудит и исправление ошибок

Проведён полный аудит кодовой базы. Выявлено и исправлено 25+ ошибок:

**Критические:**
- Deadlock семафора в AutoProvider при LLM_PROVIDER=auto (двойной захват asyncio.Semaphore)
- Race condition при проверке лимита WebSocket-соединений (неатомарная операция check-then-add)
- Блокирующие синхронные вызовы ChromaDB, Google Sheets и файлового I/O в async-контексте

**Безопасность:**
- Добавлена аутентификация для admin-эндпоинтов (ADMIN_API_KEY)
- XSS-защита в рендеринге markdown-ссылок (фильтрация javascript: URL)
- HTML-экранирование пользовательских данных в Telegram-сообщениях
- Замена детальных ошибок на generic-сообщения в voice API

**Утечки ресурсов:**
- Неограниченный рост leads_storage → добавлен MAX_LEADS_STORED
- MediaRecorder и MediaStream не освобождались при unmount компонентов
- SSE-fetch не отменялся при закрытии чата → добавлен AbortController

**Concurrency:**
- asyncio.Lock для SessionService и WebSocket лимита
- threading.Lock для TTLCache
- asyncio.Semaphore для GigaChatProvider

**Мёртвый код:**
- Удалены неиспользуемые зависимости (langchain, tiktoken, pandas, zustand и др.)
- Удалены сломанные скрипты (lint, build:widget)
- Удалён неиспользуемый параметр chat_history из build_system_prompt

## 6. Возможности AI Sale (итого)

### Для посетителей сайта:
- Мгновенные ответы на вопросы об услугах 24/7
- Голосовой ввод вопросов (микрофон)
- Озвучка ответов AI (кнопка «Слушать»)
- Быстрые кнопки для типовых вопросов
- Форма заявки (имя + телефон) прямо в чате
- Ссылки на источники в каждом ответе

### Для бизнеса (ГК Проект):
- Автоматическая квалификация лидов
- Мгновенные уведомления в Telegram-группу при новой заявке
- Дублирование заявок в Google Sheets
- Аналитика: количество сессий, популярные вопросы, конверсия
- Настраиваемый системный промпт (скрипт общения)
- Встраивание на любой сайт одной строкой (iframe)

### Технические преимущества:
- Полная контейнеризация (Docker) — деплой одной командой
- RAG — ответы только на основе реальных данных компании, без галлюцинаций
- Автоматический failover LLM (OpenAI → GigaChat)
- SSE-стриминг — ответ появляется посимвольно, как в ChatGPT
- Адаптивный дизайн — работает на десктопе и мобильных
- HTTPS + Let's Encrypt — бесплатный SSL, автообновление
